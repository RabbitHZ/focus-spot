# FocusSpot 기능 명세서

## 개요

애플워치의 건강 데이터(수면 패턴, 심박수, 호흡수 등)를 분석해 사용자의 현재 컨디션을 파악하고, 집중하기 좋은 최적의 카페를 추천하는 서비스.

---

## 기술 스택

| 레이어 | 기술 |
|--------|------|
| iOS 컴패니언 앱 | Swift / SwiftUI + HealthKit |
| 웹 프론트엔드 | Next.js (React) |
| 백엔드 API | Python FastAPI |
| 카페 데이터 | 카카오맵 API + 리뷰 크롤링 + AI 분석 |
| 데이터베이스 | PostgreSQL (카페 메타데이터) + Redis (캐시) |

---

## 핵심 기능

### 1. 건강 데이터 수집 (iOS 컴패니언 앱)

수행 역할: HealthKit 데이터를 읽어 백엔드로 전송하는 경량 앱

수집 데이터:
- 수면 시간 / 수면 단계 (깊은 수면, 렘 수면, 얕은 수면)
- 심박수 (안정 시 심박수, 최근 평균)
- 호흡수
- 혈중 산소 포화도 (SpO2)
- 활동량 / 걸음 수

동작 방식:
- 백그라운드에서 주기적으로 데이터 수집 (30분 간격)
- 앱 포그라운드 진입 시 즉시 동기화
- JWT 토큰으로 백엔드 API 인증 후 전송

---

### 2. 컨디션 분석 엔진 (백엔드 FastAPI)

수집된 건강 데이터를 기반으로 현재 상태를 5가지 모드로 분류:

| 모드 | 조건 예시 | 카페 추천 방향 |
|------|-----------|----------------|
| **집중 모드** | 수면 충분 + 심박수 정상 + 활동량 보통 | 조용하고 넓은 업무 카페 |
| **졸림 모드** | 수면 부족 + 심박수 낮음 | 약간의 소음 + 밝은 조명 카페 |
| **피로 모드** | 심박수 높음 + 수면 질 낮음 + SpO2 낮음 | 조용하고 편안한 분위기 카페 |
| **활기 모드** | 수면 충분 + 활동량 많음 + 심박수 정상 | 활기차고 사람 많은 카페 |
| **회복 모드** | 야간 활동 후 or 불규칙 패턴 | 조용하고 개인 공간 있는 카페 |

분석 로직:
- 규칙 기반 1차 분류 (가중치 점수 계산)
- 최근 7일 패턴 학습으로 개인화 보정
- 시간대 보정 (오전/오후/야간별 기준값 다름)

---

### 3. 카페 데이터베이스

#### 3-1. 데이터 수집
- 카카오맵 API: 카페 기본 정보 (위치, 영업시간, 전화번호)
- 리뷰 크롤링 대상: 카카오맵, 네이버 지도 리뷰
- AI 분석 (Claude API): 리뷰 텍스트에서 속성 추출

#### 3-2. 카페 속성 태그 (AI 추출)

| 카테고리 | 태그 예시 |
|----------|-----------|
| 소음 수준 | `quiet` / `moderate-noise` / `lively` |
| 공간 특성 | `spacious` / `cozy` / `private-booth` / `counter-seat` |
| 조명 | `bright` / `dim` / `natural-light` |
| 업무 적합도 | `work-friendly` / `no-laptop` / `power-outlet` / `fast-wifi` |
| 혼잡도 | `always-crowded` / `quiet-hours` / `easy-to-seat` |
| 음료 | `specialty-coffee` / `decaf-available` / `non-coffee` |

#### 3-3. 혼잡도 실시간 보정
- 구글/카카오 실시간 혼잡도 API 연동 (가능한 경우)
- 시간대별 평균 혼잡도 패턴 DB 저장

---

### 4. 카페 추천 알고리즘

입력값:
- 현재 컨디션 모드
- 사용자 위치 (GPS)
- 현재 시각
- 사용자 선호 필터 (선택)

추천 점수 계산:
```
score = (컨디션 매칭도 × 0.5)
      + (거리 점수 × 0.2)
      + (혼잡도 역수 × 0.2)
      + (사용자 선호도 × 0.1)
```

결과: 상위 5개 카페 + 추천 이유 텍스트

---

### 5. 웹 프론트엔드 (Next.js)

#### 메인 화면
- 현재 컨디션 상태 카드 (모드 + 간단한 설명)
- "지금 추천 카페" 버튼
- 추천 카페 리스트 (카드형)

#### 카페 카드 정보
- 카페 이름 / 거리
- 추천 이유 한 줄 (예: "지금 졸리신 것 같아요. 적당한 소음이 집중에 도움돼요")
- 소음/혼잡도 시각화 (아이콘 또는 바)
- 지도 링크 (카카오맵)

#### 내 컨디션 히스토리
- 최근 7일 컨디션 그래프
- 시간대별 집중력 패턴

#### 설정
- 위치 반경 설정 (500m / 1km / 2km)
- 선호 카페 타입 필터
- iOS 앱 연결 상태 확인

## API 설계 (FastAPI)

| 메서드 | 엔드포인트 | 설명 |
|--------|------------|------|
| POST | `/api/health/sync` | iOS 앱에서 건강 데이터 전송 |
| GET | `/api/condition/current` | 현재 컨디션 분석 결과 |
| GET | `/api/cafes/recommend` | 컨디션 기반 카페 추천 |
| GET | `/api/cafes/{id}` | 카페 상세 정보 |
| GET | `/api/condition/history` | 최근 7일 컨디션 히스토리 |
| POST | `/api/auth/register` | 회원가입 |
| POST | `/api/auth/login` | 로그인 |

## MVP 범위 (1차 버전)

포함:
- iOS 컴패니언 앱 (HealthKit → 서버 전송)
- 컨디션 분류 (5가지 모드, 규칙 기반)
- 카페 DB 구축 (서울 강남/홍대/성수 위주, 약 200개)
- 웹 추천 화면 (메인 + 카페 리스트)
- 카카오맵 API 연동

제외 (2차 이후):
- 실시간 혼잡도 연동
- 개인화 학습
- 리뷰 크롤링 자동화
- 컨디션 히스토리 그래프

---

## 데이터 흐름

```
[Apple Watch]
     ↓ HealthKit
[iOS 컴패니언 앱]
     ↓ HTTPS POST (JWT)
[FastAPI 백엔드]
     ↓ 컨디션 분석
     ↓ 카페 DB 쿼리 (PostgreSQL)
     ↓ 추천 결과 반환
[Next.js 웹]
     ↓
[사용자 화면]
```

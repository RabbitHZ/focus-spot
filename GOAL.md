# Goal: 네이버 플레이스 리뷰 스크래핑 및 키워드 추출 완성

## 목표
네이버 플레이스에서 카페 리뷰를 실제로 스크래핑해서 핵심 키워드(와이파이, 콘센트, 조용함, 업무가능, 웨이팅 등)를 정확하게 추출하고, 카페 태그에 반영되는 것을 확인한다.

## 완료 조건
- 카페 이름으로 네이버 플레이스 ID를 찾는다
- 해당 카페의 실제 방문자 리뷰 텍스트를 가져온다 (최소 5개 이상)
- 리뷰에서 키워드(와이파이, 콘센트, 조용함, 업무가능, 웨이팅, 자리여유 등)가 정확히 추출된다
- tag_source가 "review"로 표시된다 (ai_infer fallback 아님)
- 3개 이상의 서로 다른 카페에서 위 조건이 모두 통과된다

## 현재 문제
- 네이버 플레이스 페이지가 SPA 구조라 리뷰가 lazy-load됨
- `"text":"한글..."` 패턴으로 추출 시 리뷰 수가 너무 적음 (12개 한계)
- 공차, 빽다방 등 일부 카페는 HTML에 리뷰가 노출되지 않아 ai_infer로 fallback됨
- 네이버 플레이스 내부 API 엔드포인트를 찾아야 더 많은 리뷰를 가져올 수 있음

## 작업 방향
- [ ] 네이버 플레이스 방문자 리뷰 전용 API 엔드포인트 탐색 및 검증
- [ ] 리뷰 API 성공 시 페이지네이션으로 충분한 리뷰 수집 (최소 20개 이상)
- [ ] 키워드 매칭 패턴 보완 (실제 리뷰 텍스트 기반으로 정규식 튜닝)
- [ ] 3개 이상 카페에서 tag_source="review" 확인 테스트 통과
- [ ] API 서버에 반영 후 실제 추천 결과에서 태그 확인

## 참고
- 관련 파일: `api/services/review_scraper.py`
- 현재 스크래핑 방식: 네이버 플레이스 `review/visitor` 탭 HTML에서 `"text":"..."` 패턴 추출
- 성공한 케이스: 스타벅스 강남역점 (quiet, dim 태그 추출됨)
- 실패한 케이스: 공차, 빽다방 (리뷰 HTML 미노출 → ai_infer fallback)
